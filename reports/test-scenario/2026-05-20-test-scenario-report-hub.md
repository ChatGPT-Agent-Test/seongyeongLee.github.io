# 개발 작업 리포트 허브 작업 검증 테스트 시나리오

## 테스트 범위
- 메인 화면 렌더링 및 카테고리 요약
- 상세 화면 렌더링 및 조회/확인 흐름
- 체크리스트 페이지 동작
- GitHub Pages base path 반영 여부
- Supabase 연결 유무에 따른 분기 처리

## 공통 사전 조건
- `data/reports.json` 이 현재 구조와 일치한다.
- `reports/` 하위 Markdown 파일이 존재한다.
- 루트 기준으로 `index.html`, `pages`, `assets`, `data` 구조가 유지된다.
- 시간 표시는 Asia/Seoul 기준으로 해석한다.

## 시나리오 1. 메인 화면 기본 로딩
### 목적
메인 화면이 정상적으로 열리고 기본 정보가 표시되는지 확인한다.

### 절차
1. `index.html`에 접속한다.
2. 상단 제목, 설명, 마지막 갱신 정보가 표시되는지 확인한다.
3. 시스템 상태 카드가 초기 문구에서 실제 정보로 갱신되는지 확인한다.

### 기대 결과
- 화면이 깨지지 않는다.
- 루트 기준 상태 정보가 표시된다.
- 연결 상태 문구가 표시된다.

## 시나리오 2. 카테고리 요약 및 목록 동작
### 목적
카테고리 카드, 미확인/전체 개수, 목록 전환이 정상 동작하는지 확인한다.

### 절차
1. 메인 화면에서 카테고리 요약 카드를 확인한다.
2. 각 카드에 최신 보고서와 먼저 확인할 보고서가 표시되는지 확인한다.
3. 카테고리 탭을 순서대로 클릭한다.
4. 선택한 카테고리의 목록만 보이는지 확인한다.

### 기대 결과
- 4개 카테고리 카드가 보인다.
- 탭 전환 시 목록이 동적으로 바뀐다.
- 보고서 상태가 확인 완료 또는 미확인으로 표시된다.

## 시나리오 3. 상세 화면 렌더링
### 목적
상세 페이지에서 메타데이터와 Markdown 본문이 정상 렌더링되는지 확인한다.

### 절차
1. 메인 화면에서 임의의 보고서 상세 보기 링크를 클릭한다.
2. 상세 헤더, 메타 정보, 본문 영역을 확인한다.
3. Base path와 연결 상태 문구를 확인한다.

### 기대 결과
- 제목과 요약이 헤더에 표시된다.
- 카테고리, 프로젝트, 등록일, 수정일, Base path, 연결 상태가 보인다.
- Markdown 본문이 HTML로 렌더링된다.

## 시나리오 4. Supabase 미설정 분기
### 목적
Supabase 값이 설정되지 않은 상태에서도 화면이 깨지지 않는지 확인한다.

### 절차
1. `config.js`에 placeholder 값을 유지한 상태로 메인과 상세 화면을 연다.
2. 메인 화면에서 연결 상태 문구를 확인한다.
3. 상세 화면에서 확인 완료 버튼 상태를 확인한다.

### 기대 결과
- 메인 화면은 로컬 기준 안내 문구를 표시한다.
- 상세 화면은 조회 화면을 유지한다.
- 확인 완료 버튼은 비활성화되며 저장 미설정 안내를 표시한다.

## 시나리오 5. Supabase 설정 후 조회/확인 처리
### 목적
Supabase가 연결된 경우 조회 이력과 확인 처리 흐름이 동작하는지 확인한다.

### 절차
1. 유효한 `SUPABASE_URL`, `SUPABASE_ANON_KEY`를 설정한다.
2. 상세 화면에 진입한다.
3. `report_views` 테이블에 insert가 발생하는지 확인한다.
4. 확인 완료 버튼을 누른다.
5. `report_acknowledgements` upsert가 발생하는지 확인한다.
6. 메인으로 돌아가 미확인 수 변화를 확인한다.

### 기대 결과
- 조회 이력이 저장된다.
- 버튼 클릭 후 `확인 완료됨` 상태가 보인다.
- 메인 화면의 미확인 수가 감소한다.

## 시나리오 6. 체크리스트 페이지 동작
### 목적
검증 체크리스트 페이지에서 체크 상태 저장과 다음 단계 제안이 동작하는지 확인한다.

### 절차
1. `pages/root-structure-validation-checklist.html`에 접속한다.
2. 임의의 체크 항목을 선택한다.
3. 진행률과 다음 단계 제안이 바뀌는지 확인한다.
4. 브라우저 새로고침 후 체크 상태가 유지되는지 확인한다.
5. 초기화 버튼을 눌러 상태가 리셋되는지 확인한다.

### 기대 결과
- 체크 상태가 로컬 저장소에 저장된다.
- 진행률과 다음 단계가 동적으로 바뀐다.
- 초기화가 정상 동작한다.

## 시나리오 7. GitHub Pages 프로젝트 경로 검증
### 목적
프로젝트 base path 환경에서도 정적 경로가 정상 동작하는지 확인한다.

### 절차
1. `BASE_PATH='/report-hub-site'` 환경을 가정한다.
2. 메인 페이지, 상세 페이지, 체크리스트 페이지를 각각 연다.
3. CSS, JS, JSON, Markdown 로드 경로를 확인한다.

### 기대 결과
- 정적 파일 로드에 404가 발생하지 않는다.
- 상세 링크와 Markdown fetch 경로가 정상이다.
- 체크리스트 데이터 파일도 정상 로드된다.

## 우선 검증 순서
1. 메인 화면 기본 로딩
2. 상세 화면 렌더링
3. Supabase 미설정 분기
4. 체크리스트 페이지 동작
5. Supabase 설정 후 조회/확인 처리
6. GitHub Pages 프로젝트 경로 검증
