# 작업 검증 테스트 시나리오

## 작업 개요
- 작업명: GitHub Pages 루트 배포 경로 오인 원인 검증
- 작업 유형: 신규 작업
- 작성 일시: 2026-05-20

## 테스트 목적
루트 배포 저장소에서 경로 계산이 잘못되어 데이터 로딩이 실패하는 현상을 재현·확인하고, 수정 후 정상 동작 기준을 검증한다.

## 사전 조건
- 배포 저장소: `ChatGPT-Agent-Test/ChatGPT-Agent-Test.github.io`
- 서비스 주소: `https://chatgpt-agent-test.github.io/`
- 메인 페이지와 데이터 파일이 저장소 루트 기준으로 배포되어 있어야 한다.

## 테스트 시나리오

### 시나리오 1. 메인 페이지 초기 진입
- 목적: 루트 주소 접속 시 메인 레이아웃과 데이터 로딩이 모두 정상 동작하는지 확인
- 절차:
  1. 브라우저 캐시를 비우거나 강력 새로고침한다.
  2. `https://chatgpt-agent-test.github.io/` 로 접속한다.
  3. 시스템 상태, 보고서 4종 바로가기, 카테고리 요약, 보고서 목록 영역을 확인한다.
- 기대 결과:
  - 초기 플레이스홀더 문구가 실제 데이터로 교체된다.
  - 오류 문구가 노출되지 않는다.

### 시나리오 2. 데이터 파일 요청 경로 확인
- 목적: `reports.json` 요청 경로가 루트 기준으로 계산되는지 확인
- 절차:
  1. 브라우저 개발자 도구 Network 탭을 연다.
  2. 메인 페이지를 새로고침한다.
  3. `reports.json` 요청 URL을 확인한다.
- 기대 결과:
  - 요청 경로가 `/data/reports.json` 이어야 한다.
  - `/report-hub-site/data/reports.json` 형태로 요청되지 않아야 한다.

### 시나리오 3. 상세 페이지 이동 확인
- 목적: 보고서 상세 링크가 루트 배포 구조에 맞게 이동하는지 확인
- 절차:
  1. 메인 페이지에서 아무 보고서나 선택한다.
  2. 상세 페이지 URL을 확인한다.
- 기대 결과:
  - `/pages/report-detail.html?reportId=...` 형태로 이동한다.
  - 존재하지 않는 하위 프로젝트 경로로 이동하지 않는다.

### 시나리오 4. 오류 복구 확인
- 목적: 기존 오류 상태가 해소되었는지 확인
- 절차:
  1. 메인 페이지 상단 우측 상태 문구를 확인한다.
  2. 시스템 상태 카드와 보고서 목록 표시를 확인한다.
- 기대 결과:
  - `초기 로딩 중 문제가 발생했습니다.` 문구가 사라진다.
  - `시스템 상태를 불러오지 못했습니다.` 및 `데이터를 불러오지 못했습니다.` 문구가 사라진다.

## 실패 판정 기준
- 메인 화면이 보이더라도 데이터 카드가 모두 오류 문구로 남아 있으면 실패
- `reports.json` 요청이 404 또는 잘못된 하위 경로로 나가면 실패
- 상세 페이지 링크가 404를 유발하면 실패

## 비고
이번 테스트는 캐시 이슈와 별개로, 실제 코드의 경로 계산 규칙이 루트 배포 구조와 일치하는지를 확인하는 데 목적이 있다.
