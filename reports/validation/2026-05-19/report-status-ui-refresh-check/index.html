Report Status UI Refresh Check

Scenario Tests

시나리오 기반 검증

시나리오 1. 루트 허브에서 세 카테고리가 모두 보이는가

기대 결과: validation, changes, external-references 카드가 모두 노출된다.

실제 결과: 세 카테고리 카드가 모두 표시되고, 외부 참조 카테고리는 실패 색상으로 더 강하게 구분된다.

시나리오 2. 각 카테고리 인덱스에 상태와 확인 여부 배지가 함께 보이는가

기대 결과: 각 목록 항목에 상태 배지와 미확인 또는 확인 완료 배지가 함께 표시된다.

실제 결과: validation, changes, external-references 모두 같은 배지 체계로 통일되었다.

시나리오 3. 실패 보고서가 더 강한 대비로 보이는가

기대 결과: 실패 항목은 카드 배경, 보더, 그림자, 배지가 성공 항목보다 눈에 띄게 강조된다.

실제 결과: 외부 참조 카테고리와 실패 상세 페이지에 경고형 스타일이 적용되었다.

시나리오 4. 상대 링크가 GitHub Pages 구조에 맞게 유지되는가

기대 결과: 루트 허브, 카테고리 인덱스, 상세 보고서가 모두 상대 링크로 이어진다.

실제 결과: 신규 파일과 수정 파일 모두 상대 경로 기준으로 정리되었다.

Final Status
