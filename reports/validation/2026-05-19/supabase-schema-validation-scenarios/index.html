실행 시나리오

시나리오 기대 결과
연결된 프로젝트를 식별한다. 프로젝트 이름, 리전, 상태, API URL을 확인할 수 있다.
스키마 메타데이터를 읽는다. 대상 4개 테이블의 컬럼, 제약, 인덱스, 정책, 트리거가 조회된다.
초안 대비 차이를 확인한다. 필수 테이블과 핵심 컬럼, 주요 제약, 인덱스, RLS가 모두 맞으면 통과한다.
테이블을 안전하게 집계 조회한다. 쓰기 없이 각 테이블의 현재 행 수를 확인한다.
보안/성능 점검을 추가한다. Supabase advisor 결과를 별도 경고 또는 실패 항목으로 기록한다.

검증 제한 사항

이번 실행에서는 실제 사용자 인증 컨텍스트를 붙인 행 단위 정책 동작 테스트나 테스트 데이터 삽입은 수행하지 않았습니다. 현재 스키마 메타데이터와 집계 조회만으로 요구된 구조 검증은 충분히 완료했습니다.
