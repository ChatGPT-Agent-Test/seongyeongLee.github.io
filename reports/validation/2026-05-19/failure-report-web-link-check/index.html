Purpose

작업 목적

이미 GitHub에 올라가 있는 reports/preview-failure-report-2026-05-19.md를 웹에서 탐색 가능한 보고서 구조에 연결하고, reports/index.html 및 reports/validation/index.html에서 자연스럽게 이동할 수 있도록 만드는 것이 목적입니다.

Linking

실패 보고서 웹 연결 방식

  • 원본 Markdown 파일은 그대로 유지했습니다.
  • 외부 참조 카테고리 아래에 HTML 상세 페이지를 추가했습니다.
  • 루트 허브와 external-references 인덱스에서 이 상세 페이지로 이동할 수 있게 상대 링크를 연결했습니다.
  • 상세 페이지 안에서는 원본 Markdown으로 되돌아가는 링크도 함께 제공했습니다.
Pages

GitHub Pages 확인 관점

모든 링크는 reports/ 루트를 기준으로 상대 경로를 사용하도록 정리했습니다. 서버 없이 정적으로 열리는 구조라 GitHub Pages 공개 경로에서도 같은 방식으로 탐색할 수 있습니다.

이번 검증은 구조 검증 기준으로 작성했고, 실제 공개 URL 최종 확인은 저장소 반영 직후 GitHub Pages 동기화 시점에 다시 점검하면 됩니다.

Scenario Tests

시나리오 테스트 목록

시나리오 1. 루트 허브에서 external-references 인덱스로 이동

기대 결과: reports/index.html의 External References 카드가 external-references/index.html로 이동한다.

실제 결과: 루트 허브에서 해당 카테고리 카드 링크를 제공하도록 정리했다.

시나리오 2. external-references 인덱스에서 실패 보고서 HTML 상세 페이지 열기

기대 결과: 2026-05-19/preview-failure-report/index.html 상세 페이지가 열린다.

실제 결과: 외부 참조 카테고리의 목록 항목에서 상세 페이지로 연결되도록 반영했다.

시나리오 3. 실패 보고서 상세 페이지에서 원본 Markdown 열기

기대 결과: HTML 상세 페이지 안의 원본 링크가 reports/preview-failure-report-2026-05-19.md를 연다.

실제 결과: 상대 경로를 ../../../preview-failure-report-2026-05-19.md로 유지해 루트의 Markdown 원본을 가리키도록 맞췄다.

시나리오 4. 실패 성격이 목록과 상세에서 더 강하게 보이는가

기대 결과: 실패 카드와 상세 배너가 일반 성공 보고서보다 눈에 띄는 경고형 스타일을 가진다.

실제 결과: 외부 참조 카테고리와 상세 페이지에 실패 배경, 보더, 그림자를 적용했다.
