# 루트 구조 동작 점검 체크리스트

## 1. 메인 화면 점검
### 기본 로딩
- [ ] `index.html` 진입 시 화면이 깨지지 않는다
- [ ] 상단 제목, 설명, 마지막 갱신 문구가 보인다
- [ ] 초기 로딩 문구가 실제 데이터로 교체된다

### 카테고리 요약
- [ ] 4개 카테고리 카드가 모두 표시된다
- [ ] 각 카드에 `미확인 / 전체` 수치가 표시된다
- [ ] 각 카드에 최신 보고서 1건이 보인다
- [ ] 각 카드에 미확인 보고서 1건 또는 대체 문구가 보인다

### 카테고리 탭 및 목록
- [ ] 카테고리 탭 클릭 시 활성 탭이 바뀐다
- [ ] 선택한 카테고리의 보고서만 목록에 표시된다
- [ ] 각 보고서에 제목, 요약, 등록일, 수정일, 상태가 보인다
- [ ] 상세 보기 링크 클릭 시 올바른 상세 페이지로 이동한다

### 예외 처리
- [ ] `reports.json` 로드 실패 시 오류 문구가 표시된다
- [ ] 카테고리 데이터가 비어 있어도 화면이 깨지지 않는다

## 2. 상세 화면 점검
### 기본 로딩
- [ ] `pages/report-detail.html?reportId=...`로 진입 시 화면이 열린다
- [ ] 보고서 제목과 요약이 헤더에 표시된다
- [ ] 카테고리, 프로젝트, 등록일, 수정일, Base path, 연결 상태가 표시된다

### 본문 렌더링
- [ ] `file_path`에 지정된 Markdown 파일이 로드된다
- [ ] Markdown이 HTML로 렌더링된다
- [ ] 제목, 목록, 문단이 읽기 쉽게 표시된다

### 확인 처리
- [ ] 상세 진입 시 조회 이력 기록 로직이 실행된다
- [ ] 확인 완료 버튼이 보인다
- [ ] Supabase 연결 시 버튼 클릭 후 `확인 완료됨`으로 바뀐다
- [ ] Supabase 미설정 시 버튼이 비활성화되고 안내 문구가 보인다

### 예외 처리
- [ ] 잘못된 `reportId`면 보고서를 찾을 수 없다는 안내가 보인다
- [ ] Markdown 로드 실패 시 오류 문구가 표시된다

## 3. Supabase 연결 점검
### 설정 확인
- [ ] `assets/js/config.js`의 `SUPABASE_URL`이 실제 값이다
- [ ] `SUPABASE_ANON_KEY`가 실제 값이다
- [ ] placeholder 값이 아니다

### 메인 화면 연동
- [ ] `getCurrentViewer()`가 사용자 식별자를 반환한다
- [ ] `fetchAcknowledgements(viewerId)`가 정상 응답을 받는다
- [ ] acknowledgement 결과가 미확인 개수 계산에 반영된다

### 상세 화면 연동
- [ ] 상세 진입 시 `report_views` insert가 실행된다
- [ ] 확인 버튼 클릭 시 `report_acknowledgements` upsert가 실행된다

### 정책 및 권한
- [ ] anon key 기준 권한이 필요한 테이블에 맞게 설정돼 있다
- [ ] RLS 때문에 insert, upsert가 막히지 않는다
- [ ] 실패 시 브라우저 콘솔에서 오류를 확인할 수 있다

## 4. GitHub Pages base path 점검
### 경로 규칙
- [ ] `buildAppPath()`가 모든 fetch와 링크 생성에 사용된다
- [ ] 절대 경로 `/assets/...`에 직접 의존하지 않는다
- [ ] `reports.json`, Markdown, 상세 페이지 링크가 base path를 반영한다

### 프로젝트 페이지 배포 시
- [ ] 메인 페이지가 정상 로드된다
- [ ] CSS와 JS가 404 없이 로드된다
- [ ] 상세 페이지 링크가 올바른 경로로 열린다
- [ ] Markdown fetch 경로가 깨지지 않는다

### 로컬 및 초안 환경
- [ ] 로컬에서는 `BASE_PATH=''` 기준으로 정상 동작한다
- [ ] Pages 환경에서는 `BASE_PATH='/report-hub-site'` 기준으로 정상 동작한다

## 5. 데이터 파일 점검
### reports.json
- [ ] `generated_at` 값이 존재한다
- [ ] `timezone`이 `Asia/Seoul`이다
- [ ] `categories[]`의 `key`, `label`이 모두 존재한다
- [ ] 각 보고서 항목에 필수 필드가 모두 존재한다

### 보고서 파일 연결
- [ ] `file_path`가 실제 존재하는 Markdown 파일과 일치한다
- [ ] `detail_page_path`가 현재 상세 페이지 규칙과 일치한다

## 6. 최종 통합 시나리오
- [ ] 메인 진입 후 특정 보고서의 상세 화면까지 이동이 정상 동작한다
- [ ] 상세에서 확인 완료 처리 후 메인으로 돌아오면 미확인 수 감소가 반영된다
- [ ] Supabase 미설정 상태에서도 화면이 깨지지 않고 로컬 기준으로 표시된다
- [ ] GitHub Pages 프로젝트 경로에서도 정적 파일과 상세 링크가 정상 동작한다
