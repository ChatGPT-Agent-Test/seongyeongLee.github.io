# 개발 작업 리포트 허브 설계 문서

## 1. 개요
이 문서는 개발 작업 리포트 허브 에이전트가 생성·관리할 보고서 체계, 프런트엔드 페이지 구조, GitHub Pages 배포 방식, Supabase 기반 조회 이력 관리 구조를 통합적으로 정의한다.

이 시스템의 목표는 다음과 같다.
- 작업 결과를 4종 보고서 Markdown 파일로 표준화
- GitHub Pages에서 확인 가능한 메인/상세 리포트 뷰어 제공
- 여러 사용자의 보고서 확인 이력과 운영 로그를 Supabase로 관리
- 사용자 화면은 한국어, 내부 소스 구조는 영어 기반으로 일관성 유지
- 시간 기준은 항상 Asia/Seoul 사용

## 2. 현재 파일 배치 원칙
현재 Builder 구조는 다음 원칙을 따른다.
- `Integrated Agent Design System/` 폴더에는 설계 문서만 보관한다.
- 실제 실행 파일은 루트에 둔다.
- 즉 `index.html`, `pages`, `assets`, `data`, `reports`, `supabase` 는 문서 폴더 밖에서 관리한다.

## 3. 핵심 산출물
기본 보고서 카테고리는 아래 4개로 고정한다.
1. 작업 수행 계획서
2. 작업 검증 테스트 시나리오
3. 작업 검증 결과 보고서
4. 작업 내역 보고서

각 보고서는 Markdown 원본으로 보관하며, 프런트엔드에서는 이를 사용자 친화적인 HTML 화면으로 렌더링한다.

## 4. 보고서 구조 원칙
- 파일명, 폴더명, 내부 식별자, 데이터 키는 영어 기반으로 작성한다.
- 화면에 노출되는 제목, 카테고리명, 상태명, 요약 문구는 한국어를 사용한다.
- 보고서에는 공통 메타데이터와 본문 구조를 둔다.
- 최신 보고서와 미확인 보고서 우선순위를 계산할 수 있는 데이터 필드를 유지한다.

## 5. 프런트엔드 설계 원칙
### 메인 화면
메인 화면은 다음 기능을 포함한다.
- 카테고리별 요약 카드
- 카테고리별 미확인 개수 / 전체 개수 축약 표시
- 카테고리별 최신 보고서 1건 미리보기
- 카테고리별 미확인 보고서 1건 미리보기
- 카테고리 선택 탭
- 선택한 카테고리에 따른 동적 리스트 렌더링

### 상세 화면
상세 화면은 다음 기능을 포함한다.
- 선택한 보고서의 제목, 메타데이터, 요약 정보 표시
- Markdown 본문을 HTML로 파싱해 섹션별로 읽기 쉽게 재구성
- 목록 복귀, 이전/다음 보고서 이동 등 탐색 보조 기능 제공

## 6. GitHub Pages 배포 원칙
- 정적 리소스 중심 구조를 사용한다.
- 보고서 Markdown, 메타데이터 JSON, HTML/CSS/JS 파일을 함께 저장소에 배치한다.
- 상대 경로와 GitHub Pages base path를 고려해 라우팅을 구성한다.
- 사용자가 별도 저장소를 지정하지 않으면 온보딩으로 수집한 기본 저장소를 기준으로 설계한다.

## 7. Supabase 운영 원칙
- 보고서 메타정보와 조회 이력, 확인 상태, 운영 로그를 구분된 테이블로 관리한다.
- 미확인 상태는 사용자별 acknowledgement 존재 여부로 계산한다.
- 조회 이벤트는 별도 로그성 데이터로 축적한다.
- 시간 저장은 표준 timestamptz를 사용하되 사용자 노출 시 한국 시간으로 변환한다.

## 8. 병렬 작업 분리 기준
이 시스템은 다음 단위로 동시에 작업할 수 있게 설계한다.
- 보고서 템플릿 설계
- 프런트엔드 화면 설계
- 정적 데이터/메타데이터 구조 설계
- Supabase 스키마 및 API 구조 설계

병렬 작업 후에는 반드시 아래 항목의 정합성을 검토한다.
- category key 일치 여부
- report_id와 파일 경로 연결
- 프런트엔드 렌더링 대상 데이터 구조
- Supabase 조회/확인 상태 계산 로직
- GitHub Pages 배포 경로

## 9. 시간/언어 정책
- 기본 시간대: Asia/Seoul
- UI 언어: 한국어
- 내부 코드/DB/API/파일 구조: 영어

## 10. 구현 권장 순서
1. 보고서 템플릿 작성
2. reports metadata JSON 구조 확정
3. 메인/상세 페이지 HTML 초안 작성
4. CSS/JS 유틸 작성
5. Supabase DDL 작성
6. GitHub Pages 배포 연결
